# Atom Generator

`AtomGenerator` is the alpha0 provisioning facade for creating new atoms. It turns atom birth from a loose sequence of primitive calls into one governed entrypoint:

```text
atm create -> generateAtom() -> allocateAtomId() -> scaffoldAtomWorkbench() -> runAtomicTestRunner() -> createAtomicRegistryEntry() -> writeRegistryArtifacts()
```

## Responsibilities

- Allocate canonical IDs with `ATM-{BUCKET}-{NNNN}` format.
- Initialize a minimal valid Atomic Spec.
- Delegate workbench file creation to the existing scaffold builder.
- Run the declared validation command through the existing test runner.
- Register the atom in `atomic-registry.json` and refresh the registry catalog.

The generator is intentionally a facade. It does not redefine the parser, atom-space layout, scaffold builder, hash-lock, or registry semantics.

## API

```js
import { generateAtom } from './packages/core/src/manager/atom-generator.mjs';

const result = generateAtom({
  bucket: 'FIXTURE',
  title: 'Example Atom',
  description: 'Proof atom generated through the provisioning facade.',
  logicalName: 'atom.fixture-example'
});
```

The result includes `atomId`, `workbenchPath`, `specPath`, `testPath`, `allocation`, and per-phase execution records. Repeating the same `logicalName` is idempotent and returns the existing registry entry.

## CLI

```bash
node packages/cli/src/atm.mjs create \
  --bucket FIXTURE \
  --title ExampleAtom \
  --description "Proof atom generated through the provisioning facade." \
  --logical-name atom.fixture-example
```

Use `--dry-run` to preview allocation and planned files without writing to disk.

## Self Governance

The generator itself is registered as `ATM-CORE-0004` with logicalName `atom.core-atom-generator`.

```text
atomic_workbench/atoms/ATM-CORE-0004/
  atom.spec.json
  atom.test.ts
  atom.test.report.json
```

This keeps the provisioning facade under the same registry, hash-lock, and evidence rules as the atoms it creates. The CLI command is only a host facade; the core behavior lives in `packages/core/src/manager/atom-generator.mjs` and `packages/core/src/manager/id-allocator.mjs`.

## Dogfood Evidence

`ATM-FIXTURE-0001` was generated by `atm create` as dogfood evidence. It proves the alpha0 generator can allocate an ID, scaffold a workbench, run validation, register the entry, and refresh the catalog.

## Alpha0 Limits

- Sequence allocation is registry-based max+1; it does not use a reservation file yet.
- Generated fixture atoms are spec/test/report based. Dedicated source-file templates are a follow-up task.
- Evolution of existing atoms is out of scope; this facade handles birth only.
