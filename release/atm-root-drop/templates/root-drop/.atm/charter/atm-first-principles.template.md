# ATM First Principles - {{PROJECT_NAME}}

**Charter version**: {{CHARTER_VERSION}}
**Last amended**: {{LAST_AMENDED_DATE}}
**Schedule**: Schedule A - economic thresholds

ATM governance exists to preserve delivery truth while keeping coordination cost bounded.

## Non-Negotiable Principles

1. Agents must not mutate files outside their governed scope.
2. Delivery must pass the validators declared by the active task contract.
3. Evidence must correspond to the final delivered content, not an earlier draft.
4. Shared branches must serialize commit and push authority.
5. Governance overhead must stay economically bounded.
6. Team Agents must prove measured acceleration in actual money and end-to-end time.
7. Raw token counts are mandatory diagnostics, but raw tokens are not a cross-model price proxy.
8. Normal parallel development uses one canonical worktree/base/HEAD; a physical
   file overlap is evaluated from bounded logical intent and can compose through
   a neutral steward rather than becoming a path-only lock.
9. Git branches, worktrees, indexes, merges, and rebases are outer delivery
   mechanisms, not normal ATM concurrency or isolation mechanisms.

## Schedule A - Economic Thresholds

Production governance must not exceed unmanaged work by more than 10 percent in actual monetary cost or end-to-end time.

Preferred governance should stay under 5 percent overhead.

Breakthrough Team Agent usage should reduce end-to-end time while staying within the configured production cost cap.

If a Team Agent run cannot show measured acceleration, ATM should scale down the crew or return to a single-agent lane and record the reason as optimization evidence.
