# Codex Entry-Only Graduation Plan

Generated: 2026-05-26  
Generated by: TASK-ASA-0012

## Current State: Entry-Only

The Codex integration adapter is currently in **entry-only** status, meaning:

- The Codex skill harness recognises ATM commands and can route to them
- Codex agents can read and invoke `node atm.mjs <command>`
- Codex does **not** auto-execute pre-commit-style gates inside the conversation
- ATM relies on local git hooks (configured by `bootstrap-entry`) to enforce gates when Codex commits

## Why Entry-Only

Codex skill profiles run as user-invoked commands rather than always-on hooks. Without an always-on hook integration, ATM cannot guarantee that an in-conversation Codex agent will run `guard atomization-coverage` before suggesting a commit. The local git pre-commit hook still catches violations at commit time, but this is a later enforcement point than for `claude-code` or `cursor` adapters which can enforce within the agent loop.

## Graduation Criteria

The Codex adapter graduates from **entry-only** to **active** when:

1. Codex skill harness exposes an `on-pre-commit` or `on-stage` hook that runs without user re-invocation
2. ATM `codex` adapter ships a skill manifest that:
   - Registers for the new hook
   - Wraps `node atm.mjs guard atomization-coverage` and `node atm.mjs guard atom-callsite-readability`
   - Reports skill exit code back to the Codex harness so it can block the suggestion
3. Self-hosting alpha test passes with Codex enforcing both guards inline
4. `validate:integration-adapter` validator passes for `codex` with `enforcement: active` instead of `enforcement: entry-only`

## Interim Enforcement

Until graduation:

- Codex agents are instructed via skill manifest to run `node atm.mjs guard atomization-coverage --new-paths <files>` after staging new production sources
- Local git pre-commit hook runs the atomization-coverage guard regardless of which agent staged the files
- `validate:atomization-coverage` is in the standard validator suite; CI failure flags any drift

## Exclusion Alternative

If Codex skill harness never supports always-on hooks, the alternative is to:

1. Mark the Codex adapter as `enforcement: entry-only` permanently in `validator-to-atom-relationship.json`
2. Document the gap in `docs/INTEGRATION_ADAPTER_LIMITATIONS.md`
3. Continue relying on git pre-commit hook as the final enforcement point

## Tracking

Progress on this graduation plan is tracked in:

## Notes

- The ATM framework remains usable with Codex; the only difference is enforcement timing
- Graduation will become more urgent if Codex adoption grows beyond solo developer workflows
